Gemini의 응답
기존 network 모듈을 없애고 내용을 분산시킨다면, VPC와 퍼블릭 네트워크는 aws 모듈로, 모니터링 전용 프라이빗 네트워크는 monitoring 모듈로 이동하는 것이 가장 논리적입니다.

하이브리드 클라우드 구조에서 AWS 측 인프라의 중심(Bastion, k3s Standby)이 aws 모듈에 있으므로, 이 모듈이 VPC라는 '바구니'를 관리하게 하고 monitoring 모듈은 그 바구니 안에 자신의 공간(Subnet)만 만드는 방식입니다.

이동해야 할 파일별 상세 내용은 다음과 같습니다.

1. modules/aws/로 이동할 내용 (VPC 및 퍼블릭망)
AWS의 기본 뼈대와 외부 통신을 담당하는 리소스들이 이곳으로 들어갑니다.

modules/aws/network.tf (신규 작성 또는 추가)

Terraform
# VPC 생성 (AWS 인프라의 시작점)
resource "aws_vpc" "main" {
  cidr_block           = var.vpc_cidr
  enable_dns_hostnames = true
  enable_dns_support   = true

  tags = {
    Name        = "${var.project_name}-${var.environment}-vpc"
    Project     = var.project_name
    Environment = var.environment
  }
}

# 인터넷 관문 (IGW)
resource "aws_internet_gateway" "main" {
  vpc_id = aws_vpc.main.id

  tags = {
    Name        = "${var.project_name}-${var.environment}-igw"
  }
}

# 퍼블릭 서브넷 (Bastion, k3s 노드용)
resource "aws_subnet" "public" {
  vpc_id                  = aws_vpc.main.id
  cidr_block              = var.public_subnet_cidr
  availability_zone       = var.availability_zone
  map_public_ip_on_launch = true

  tags = {
    Name = "${var.project_name}-${var.environment}-public-subnet"
    Tier = "public"
  }
}

# 라우팅 테이블 (인터넷 통신 설정)
resource "aws_route_table" "public" {
  vpc_id = aws_vpc.main.id
  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.main.id
  }
}

resource "aws_route_table_association" "public" {
  subnet_id      = aws_subnet.public.id
  route_table_id = aws_route_table.public.id
}
modules/aws/outputs.tf (추가)
다른 모듈(monitoring)에서 VPC 정보를 참조할 수 있도록 출력해야 합니다.

Terraform
output "vpc_id" {
  value = aws_vpc.main.id
}

output "vpc_cidr" {
  value = aws_vpc.main.cidr_block
}

output "public_subnet_id" {
  value = aws_subnet.public.id
}
2. modules/monitoring/로 이동할 내용 (프라이빗 서브넷)
모니터링 서버가 위치할 전용 공간입니다.

modules/monitoring/network.tf (신규 작성)
aws 모듈에서 만든 vpc_id를 전달받아 서브넷을 생성합니다.

Terraform
# 프라이빗 서브넷 (모니터링 전용)
resource "aws_subnet" "private" {
  vpc_id            = var.vpc_id # aws 모듈에서 전달받은 값
  cidr_block        = var.private_subnet_cidr
  availability_zone = var.availability_zone
  map_public_ip_on_launch = false

  tags = {
    Name = "${var.project_name}-${var.environment}-private-subnet"
    Tier = "private"
  }
}
3. 루트 infra/terraform/main.tf에서의 연결
공통 network 모듈 호출은 삭제하고, aws 모듈의 아웃풋을 monitoring 모듈의 인풋으로 꽂아줍니다.

Terraform
# 1. AWS 모듈 (VPC 및 퍼블릭망 구축)
module "aws" {
  source             = "./modules/aws"
  project_name       = var.project_name
  environment        = var.environment
  vpc_cidr           = var.vpc_cidr
  public_subnet_cidr = var.public_subnet_cidr
  availability_zone  = var.availability_zone
  # ...
}

# 2. 모니터링 모듈 (AWS가 만든 VPC 안에 서브넷 구축)
module "monitoring" {
  source              = "./modules/monitoring"
  project_name        = var.project_name
  environment         = var.environment
  
  # 데이터 연결의 핵심
  vpc_id              = module.aws.vpc_id 
  private_subnet_cidr = var.private_subnet_cidr
  availability_zone   = var.availability_zone
  # ...
}
요약 및 주의사항
VPC는 하나: network 모듈을 없애더라도 실제로는 하나의 VPC를 공유해야 하므로, 리소스의 소유권(VPC 생성)은 aws 모듈이 갖고 monitoring은 이를 빌려 쓰는 구조가 됩니다.

의존성: 테라폼은 module.aws.vpc_id를 참조하는 순간, aws 모듈을 먼저 배포한 후 monitoring 모듈을 배포하도록 순서를 자동으로 조율합니다.

변수 관리: 기존 network 모듈에서 쓰던 vpc_cidr, subnet_cidr 변수들은 이제 aws와 monitoring 모듈의 각 variables.tf에 정의되어야 합니다.

이제 이 구조대로 파일을 찢어서 배치하시면 됩니다. aws 모듈의 variables.tf에 추가해야 할 전체 변수 목록이 필요하신가요?

    Tier = "private"
  }
}
3. infra/terraform/main.tf (최종 통합 오케스트레이션)
이제 루트 파일에서 클라우드플레어 설정을 통괄하고 각 모듈을 연결합니다.

Terraform
# 1. 클라우드플레어 (모든 터널 예약 및 토큰 생성)
module "cloudflare" {
  source                = "./modules/cloudflare"
  cloudflare_account_id = var.cf_account_id
  cloudflare_zone_id    = var.cf_zone_id
  tunnel_secret         = var.cf_tunnel_secret
  project_name          = var.project_name
  environment           = var.environment
}

# 2. AWS 모듈 (VPC 생성 및 Standby 인프라)
module "aws" {
  source             = "./modules/aws"
  project_name       = var.project_name
  environment        = var.environment
  vpc_cidr           = var.vpc_cidr
  public_subnet_cidr = var.public_subnet_cidr
  availability_zone  = var.availability_zone
  
  # 클라우드플레어 터널 토큰 주입
  tunnel_token       = module.cloudflare.aws_tunnel_token
}

# 3. Monitoring 모듈 (AWS VPC 내에 전용 공간 구축)
module "monitoring" {
  source              = "./modules/monitoring"
  project_name        = var.project_name
  environment         = var.environment
  
  # AWS 모듈에서 생성된 VPC ID 전달 (의존성 연결)
  vpc_id              = module.aws.vpc_id
  private_subnet_cidr = var.private_subnet_cidr
  availability_zone   = var.availability_zone
  
  # 클라우드플레어 터널 토큰 주입
  tunnel_token        = module.cloudflare.monitoring_tunnel_token
  
  # 보안 그룹 연동
  bastion_sg_id       = module.aws.bastion_sg_id
}

# 4. GCP 모듈 (Primary 인프라)
module "gcp" {
  source       = "./modules/gcp"
  project_id   = var.gcp_project_id
  tunnel_token = module.cloudflare.gcp_tunnel_token
  # ... 기타 설정
}
