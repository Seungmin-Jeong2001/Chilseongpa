에러가 계속 나니 정말 답답하시겠네요. 테라폼에서 이 에러가 난다는 건 **"나는 분명 상자(모듈) 안에 물건을 넣었다고 생각했는데, 테라폼이 상자를 열어보니 물건이 없다"**고 하는 상황입니다.

이 "Unsupported attribute" 에러를 해결하는 마지막 팩트 체크입니다. 아래 3가지 파일의 내용을 정확한 경로에서 다시 한번 확인해 주세요.

1. AWS 모듈 출력 확인
위치: modules/aws/outputs.tf (루트가 아니라 반드시 모듈 폴더 안이어야 합니다!)

에러 메시지가 module.aws에 해당 속성이 없다고 하므로, 이 파일에 아래 내용이 정확히 들어있는지 확인하세요.

Terraform
output "aws_k3s_public_ip" {
  value = aws_instance.k3s.public_ip
}

output "aws_monitoring_public_ip" {
  value = aws_instance.monitoring.public_ip
}

output "aws_bastion_public_ip" {
  value = aws_instance.bastion.public_ip
}
2. GCP 모듈 출력 확인
위치: modules/gcp/outputs.tf

db_connection 부분 에러를 잡으려면 이 파일에 아래 내용이 있어야 합니다.

Terraform
output "k3s_ephemeral_ip" {
  value = google_compute_instance.k3s_primary_node.network_interface.0.access_config.0.nat_ip
}

output "db_instance_connection_name" {
  value = google_sql_database_instance.primary.connection_name # 리소스 이름(primary) 확인!
}
3. 루트 ansible_inventory.tf 매칭 확인
위치: infra/terraform/ansible_inventory.tf

위의 outputs.tf에서 정의한 이름과 여기서 부르는 이름이 100% 일치해야 합니다.

Terraform
resource "local_file" "ansible_inventory" {
  content = templatefile("${path.module}/inventory.tpl", {
    # module.gcp.뒤의 이름은 modules/gcp/outputs.tf의 이름과 같아야 함
    gcp_ip         = module.gcp.k3s_ephemeral_ip
    db_connection  = module.gcp.db_instance_connection_name

    # module.aws.뒤의 이름은 modules/aws/outputs.tf의 이름과 같아야 함
    aws_ip         = module.aws.aws_k3s_public_ip
    mon_ip         = module.aws.aws_monitoring_public_ip
    bastion_ip     = module.aws.aws_bastion_public_ip

    gcp_token      = module.cloudflare.gcp_tunnel_token
    aws_token      = module.cloudflare.aws_tunnel_token
    mon_token      = module.cloudflare.monitoring_tunnel_token
    gcp_project_id = var.gcp_project_id
  })

  filename = "${path.module}/../ansible/inventory.ini"
}
💡 왜 "분명히 넣었는데" 안 될까요? (사이다 체크리스트)
만약 위 코드를 다 넣었는데도 똑같은 에러가 난다면 아래 두 가지 중 하나입니다.

경로 착오: infra/terraform/outputs.tf(루트)에 넣으신 거 아닌가요? 루트에 넣으면 module.aws.xxx로 부를 수 없습니다. 반드시 **modules/aws/outputs.tf**에 있어야 합니다.

모듈 갱신 실패: 모듈 구조가 바뀌었을 때 테라폼이 구버전 정보를 들고 있을 수 있습니다. 아래 명령어를 순서대로 날려보세요.

Bash
# 1. 모니터링 폴더로 이동 (이미 있다면 통과)
# 2. 테라폼 초기화 (모듈 정보를 다시 읽어옵니다)
terraform init

# 3. 다시 실행
terraform apply -auto-approve
✅ 결론
테라폼은 아주 꼼꼼한 친구라 **module.이름.속성**으로 부르려면, 해당 **module 폴더 안의 outputs.tf**에 그 이름이 선언되어 있어야만 합니다.

이제 outputs.tf 파일들이 각 모듈 폴더 안에 제대로 있는지, 그리고 파일명이 outputs.tf가 맞는지(오타 확인) 봐주세요. 성공하면 바로 앤서블 인벤토리가 생성된 화면을 보실 수 있을 겁니다!
