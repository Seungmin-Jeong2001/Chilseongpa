Gemini의 응답
네, 당연히 가능합니다! 앤서블의 가장 강력한 기능이 바로 **'그룹(Groups)'**입니다.

하나의 플레이북(setup.yml) 안에 여러 개의 **'Play(공연)'**를 정의하면 됩니다. 마치 연극에서 배역마다 대사가 다르듯, 앤서블도 **"GCP 너는 이거 해", "모니터링 너는 저거 해"**라고 지정할 수 있습니다.

🛠️ 앤서블 "배역" 나누기 (통합 플레이북)
아래 코드는 하나의 파일이지만, hosts 설정을 통해 각 서버에 맞는 패키지만 골라 설치합니다.

1. 인벤토리 설정 (inventory.ini)
먼저 서버들을 그룹으로 묶어줍니다.

Ini, TOML
[gcp]
gcp-main ansible_host=GCP_IP

[aws]
aws-sub ansible_host=AWS_IP

[monitoring]
monitor-server ansible_host=MONITOR_IP

[all:vars]
ansible_user=ubuntu
ansible_ssh_private_key_file=../terraform/gcp_key
2. 통합 플레이북 (setup.yml)
이 파일 하나를 실행하면 앤서블이 위 인벤토리를 보고 알아서 찾아갑니다.

YAML
---
# ---------------------------------------------------------
# Play 1: 모든 서버 공통 (Docker & Cloudflared)
# ---------------------------------------------------------
- name: "공통 환경 구축"
  hosts: all
  become: yes
  tasks:
    - name: "도커 및 필수 패키지 설치"
      shell: "curl -fsSL https://get.docker.com | sh"
      args:
        creates: /usr/bin/docker

    - name: "Cloudflared 설치"
      apt:
        deb: "https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb"

# ---------------------------------------------------------
# Play 2: GCP 메인 서버 전용 (DB 연결용 Proxy)
# ---------------------------------------------------------
- name: "GCP 서버 전용 설정"
  hosts: gcp
  become: yes
  tasks:
    - name: "Cloud SQL Proxy 컨테이너 실행"
      docker_container:
        name: cloud-sql-proxy
        image: gcr.io/cloud-sql-connectors/cloud-sql-proxy:latest
        restart_policy: always
        command: "--address 0.0.0.0 --port 3306 chilseongpa:region:db-name"

# ---------------------------------------------------------
# Play 3: AWS 모니터링 서버 전용 (Prometheus & Grafana)
# ---------------------------------------------------------
- name: "모니터링 서버 전용 설정"
  hosts: monitoring
  become: yes
  tasks:
    - name: "모니터링 도구(Prometheus/Grafana) 실행"
      community.docker.docker_compose_v2:
        project_src: /home/ubuntu/monitoring
        definition:
          services:
            prometheus: { image: prom/prometheus, ports: ["9090:9090"] }
            grafana: { image: grafana/grafana, ports: ["3000:3000"] }
💡 왜 이렇게 하면 각각 다르게 설치되나요?
앤서블은 플레이북을 위에서부터 읽으면서 hosts 조건을 확인합니다.

hosts: all: 인벤토리에 적힌 모든 서버에 접속해서 실행합니다. (공통 설치)

hosts: gcp: 인벤토리에서 [gcp] 그룹에 속한 서버만 골라서 실행합니다. (GCP 전용)

hosts: monitoring: 마찬가지로 [monitoring] 그룹 서버에만 명령을 내립니다.

🚩 실행 전 체크리스트
파일 경로: ansible-playbook -i inventory.ini setup.yml 명령어를 실행하는 위치에 두 파일이 다 있어야 합니다.

변수 주입: 각 서버마다 다른 터널 토큰은 inventory.ini의 각 호스트 옆에 tunnel_token=... 식으로 적어주면 플레이북에서 {{ tunnel_token }}으로 꺼내 쓸 수 있습니다.
